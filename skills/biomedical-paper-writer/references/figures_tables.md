# 图表规范指南

基于 khufkens/paper_writing_checklist。

---

## Figures（图）

### 设计原则

1. **可读性**
   - 确保打印和屏幕显示都清晰
   - 所有文字足够大 (最小 8pt)
   - 线条足够粗

2. **坐标轴**
   - 所有坐标轴必须有标签
   - 包含单位
   - 刻度清晰

3. **颜色使用**
   - 优先使用符号和线型而非颜色
   - 必须使用颜色时选择色盲友好配色
   - 推荐使用 [ColorBrewer](http://colorbrewer2.org/) 配色方案
   - 保持多图之间配色一致

4. **图例说明**
   - 图例标题解释图的目的和内容
   - 无需阅读正文即可理解图的含义
   - 包含必要的统计信息和样本量

### 图的类型选择

| 数据类型 | 推荐图类型 |
|----------|-----------|
| 比较分类数据 | 柱状图 |
| 展示趋势/时间序列 | 折线图 |
| 相关性 | 散点图 |
| 分布 | 箱线图、直方图 |
| 比例 | 饼图（谨慎使用） |

### 常见问题

- ❌ 3D 效果（除非数据本身是 3D）
- ❌ 过度装饰
- ❌ 图例与数据重叠
- ❌ 过小或过大的误差线

## Tables（表）

### 设计原则

1. **清晰性**
   - 表格标题解释内容和目的
   - 列标题清晰，包含单位
   - 使用脚注解释缩写

2. **格式**
   - 数字对齐（通常右对齐）
   - 小数位数一致
   - 适当使用空行分隔不同组

3. **统计信息**
   - 包含必要的统计量（均值、SD、n 等）
   - 标注显著性水平
   - 解释使用的统计检验

### 表格 vs 图

- **选择表格**: 展示精确数值、多变量比较
- **选择图**: 展示趋势、模式、关系

---

## 图表数量控制

- 主文限制图表数量（通常 6-8 个组合）
- 补充材料可包含额外图表
- 避免重复呈现相同数据

---

## 图表检查清单

### Figure 检查
- [ ] 分辨率足够 (通常 ≥300 dpi)
- [ ] 所有轴有标签和单位
- [ ] 色盲友好
- [ ] 图例解释完整
- [ ] 在正文中按顺序引用

### Table 检查
- [ ] 标题描述清晰
- [ ] 列标题有单位
- [ ] 缩写有解释
- [ ] 数值精确一致
- [ ] 在正文中按顺序引用
